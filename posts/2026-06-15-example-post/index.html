Example Post: Every Supported Element

By ylnhari · June 15, 2026 · reference, agents, blogging

This post is a living reference for AI agents writing to this blog. Every element shown here is supported by both GitHub Pages and Medium's importer. Use it as a copy-paste starting point.

Text Formatting

Use bold for key terms and emphasis. Use italic for titles, foreign words, or soft emphasis. Use inline code for short code references like function names or CLI flags. Combine them: bold italic for maximum emphasis (use sparingly).

Links

All links must use absolute URLs. Internal blog links: ylnhari.github.io/blog. External: medium.com/@ylnhari. Never use relative paths like ../other-post/.

Lists

Unordered list — use for non-sequential items:

  • First bullet point with bold emphasis
  • Second bullet with a link
  • Third bullet with inline code

Ordered list — use for steps or rankings:

  1. Install dependencies with pip install -r requirements.txt
  2. Configure your environment variables
  3. Run the server: python main.py

Blockquotes

Use blockquotes for pull quotes, key insights, or attributions:

"The best code is no code at all. Every new line you willingly bring into the world is code that has to be debugged, code that has to be read and understood." — Jeff Atwood

Code Blocks

Always wrap multi-line code in <pre><code>. Medium imports these as fenced code blocks. Note: use HTML entities for angle brackets inside code: &lt; and &gt;.

import anthropic

client = anthropic.Anthropic()

def generate_blog_post(topic: str) -> str:
    """Generate a blog post using Claude."""
    message = client.messages.create(
        model="claude-opus-4-8",
        max_tokens=4096,
        messages=[
            {
                "role": "user",
                "content": f"Write a technical blog post about: {topic}"
            }
        ]
    )
    return message.content[0].text

# Example usage
post = generate_blog_post("running LLMs locally with vLLM")
print(post)

Images

Images must use absolute HTTPS URLs. Medium downloads and re-hosts them during import, so the source URL only needs to be live at import time.

Preferred image sources in order of preference:

  1. Unsplash — https://images.unsplash.com/photo-{ID}?w=1200&q=80
  2. Wikimedia Commons — always check the license
  3. This repo's assets — store in posts/YYYY-MM-DD-slug/images/ and reference via raw GitHub URL

Section Separators

Use <hr> to create a visual break between major sections. The stylesheet renders it as "· · ·". Use at most 2 per post.

Content after the separator continues here.

YouTube Videos

Medium doesn't support <iframe> or <video> tags. To embed a YouTube video, put the bare URL in a paragraph. After importing to Medium, click the URL in the editor and Medium auto-converts it to an embedded player.

https://www.youtube.com/watch?v=dQw4w9WgXcQ

What NOT to Use

These elements break Medium's importer or cause save errors. Never use them:

  • No JavaScript — <script> tags are stripped
  • No CSS — <style> tags or style="..." attributes are stripped
  • No iframes — <iframe> is stripped (use bare YouTube URLs instead)
  • No tables — Medium has no table support; use lists instead
  • No relative URLs — all <img src> must be absolute HTTPS
  • No H4–H6 — Medium only supports H1, H2, H3
  • No div or span — use semantic HTML only
